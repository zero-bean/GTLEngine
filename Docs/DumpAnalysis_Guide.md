# 덤프 파일 분석 가이드

## 다른 PC에서 덤프 파일 받았을 때

### 매우 간단합니다!

1. `Saved/Crashes` 폴더 받기
   - `Crash_xxx.dmp` (덤프 파일)
   - `Mundi.exe` (실행 파일, 자동 포함됨)

2. Visual Studio로 덤프 열기
   - **File → Open → File** → 덤프 파일 선택
   - **"Debug with Native Only"** 버튼 클릭

3. 끝! 자동으로 스택 트레이스와 변수 값 확인 가능

**추가 작업 필요 없음** - 크래시 발생 시 exe가 자동으로 함께 저장됩니다.

### Visual Studio 심볼 설정 확인

**도구 → 옵션 → 디버깅 → 기호**에서:
- ✅ `\\172.21.11.95\SymbolServer` 추가
- ✅ `C:\SymCache` (또는 로컬 캐시 경로) 추가

---

## 문제 해결

### "일치하는 이진 바이너리 파일이 없습니다"

**원인**: 오래된 덤프 파일 (exe 자동 복사 기능 추가 전에 생성됨)

**해결**: `Saved/Crashes` 폴더에 `Mundi.exe`를 수동으로 복사

### "심볼을 로드할 수 없습니다"

**원인**: Visual Studio 심볼 경로 설정 누락

**해결**:
1. 도구 → 옵션 → 디버깅 → 기호
2. `\\172.21.11.95\SymbolServer` 추가
3. 덤프 파일 다시 열기

### 심볼 서버 접근 불가

**원인**: 네트워크 연결 끊김 또는 서버 PC 꺼짐

**해결**:
- 로컬 캐시(`C:\SymCache`)에 이미 다운로드된 심볼이 있으면 사용 가능
- 서버 PC가 켜져 있는지 확인: `ping 172.21.11.95`

---

## 요약

**협업자가 덤프 파일 받았을 때**:
1. ✅ `Saved/Crashes` 폴더 받기
2. ✅ Visual Studio로 `.dmp` 파일 바로 열기
3. ✅ 끝!

**완전 자동화** - 크래시 발생 시 exe가 자동으로 포함됩니다.
